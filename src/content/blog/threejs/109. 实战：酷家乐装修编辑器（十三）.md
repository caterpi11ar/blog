上节我们把 2D 视图画出来了：

![image.png](/assets/threejs/fdb6972a681f02d1c1546df29a7c252099e5bf00.png)

但它有些偏差。

为啥会有偏差呢？

用的同一份数据，3D 视图对了那应该 2D 视图也一样才对。

先用那个简单的户型，注释掉地板，看一下：


![image.png](/assets/threejs/53da4a842487bd0244d9bf57a38fc94f95ff3910.png)


![image.png](/assets/threejs/9545e641f9bc72e2ef83226c99335e3310f4de87.png)

现在是有偏差的，但你把每面墙旋转 180 度：


![image.png](/assets/threejs/b65edbc1d07654fd8e5f7aab82692d9298a3c458.png)


![image.png](/assets/threejs/2a3e86fd35253cff493059e9cdf441d7887ff976.png)

你会发现它们虽然位置不对，但是之前的缺口能对上了。

也就是说之前的问题在于画出的 Shape 正好是反着的，只要把它翻转下就好了。

形状反了，那位置也得反过来才行。


![image.png](/assets/threejs/c224ba821f6343dbf702870558458ae39ca6c69d.png)

```javascript
wall.position.set(-item.position.x, -item.position.y, -item.position.z);
```
设置相反的位置。

看下效果：


![2025-06-26 15.02.23.gif](/assets/threejs/770a1b0ad4dac74cce007d6d378e62bcb97735c3.gif)

现在就对上了。

也就是说，**当画出的 Shape 反着的时候，可以让它旋转 180 度，位置设置相反数，这样就正过来了。**

当然，另一种方式就是改一下绘制逻辑，比如本来是第一种顺序，改成第二种自然就反过来了：


![image.png](/assets/threejs/fa4e839656146e5ee1e328a0b49be454b4ecbf8d.png)

这里我们绘制逻辑就不改了，翻转一下就行。

![image.png](/assets/threejs/61cc5fc7468b10bfc92b4a2a5c0eb3b371bcf334.png)

![2025-06-26 15.10.14.gif](/assets/threejs/8c6ad69d4a7709cc0d48de2bdb1507376ed6a106.gif)

我们改一下数值：

![image.png](/assets/threejs/21f9c9fc4de08bd2fd8fba617bc4e67fd968f345.png)

```javascript
points: [
    { x: 0, z: 0},
    { x: 0, z: 6800},
    { x: 4800, z: 6800},
    { x: 4800, z: 0},
    { x: 0, z: 0 }
]
```
就是把墙面的 200 厚度留出来。

![2025-06-26 15.20.02.gif](/assets/threejs/9bcbcbae9032fc07e445a406441e230ab6a178b4.gif)

## 总结

这节我们修复了 2D 视图绘制时的误差。

有两个原因：

一个是 2D 视图绘制的形状是反着的，所以我们要旋转 180 度，然后把位置设置相反数，这样就反过来了。

另一个之前绘制地板没有把墙的厚度考虑进度，我们调整了下地板的点，还有绘制的起始位置就好了。

2D 视图的问题修复了，下节继续加上门窗、尺寸的绘制。
