![2025-07-07 21.07.34.gif](/assets/threejs/1a4cc7d2db61bea3367247bab0ac848b443952a3.gif)

点击切换户型后没反应，刷新之后才会生效。

这很明显，是我们加上的这段代码导致的：


![image.png](/assets/threejs/1c295ead3d9cc843fd7de53547575294f7c074b8.png)

只有 walls 为空数组的时候才会销毁绘制的房子。

但切换户型的时候，我们是直接换了另一个户型数据：

![image.png](/assets/threejs/035a10cd874fd8977cbf5d9f76c0e9a0e4a5ef0e.png)

根本没有清空。

所以要加一下清空逻辑：



![image.png](/assets/threejs/f2f693b02dbdeb42062dc3cbccd938b26ad91d69.png)


![image.png](/assets/threejs/45c3bb3b0b18dbc7998d24875dcb09be51124c93.png)
```javascript
onConfirm={() => {
    setData({
        walls: [],
        floors: [],
        ceilings: [],
        furnitures: []
    });
    setTimeout(() => {
        setData(data1);
    }, 0);
}}
```
```javascript
onConfirm={() => {
    setData({
        walls: [],
        floors: [],
        ceilings: [],
        furnitures: []
    });

    setTimeout(() => {
        setData(data2);
    }, 0 );
}}
```
改成点击切换户型的时候，先清空，然后一秒之后设置新的数据。

当墙的数组为空的时候，也不渲染：


![image.png](/assets/threejs/33cccaae266b6bbb1ad4ed00a24a2745722f273a.png)

```javascript
if(!data.walls.length) {
    return;
}
```
2D、3D 的都要改下。

试下效果：


![2025-07-08 01.00.53.gif](/assets/threejs/8aeeb653742857ac765bec50302cf0cea077956e.gif)

户型可以切换了，但现在墙可见性的计算不对了：


![2025-07-08 01.01.51.gif](/assets/threejs/2961bdc214517b692e22a08776a160cdc26b9b0a.gif)

这是为什么呢？

![image.png](/assets/threejs/8b7abbc340d4b8bc5fc38d9553ac348a14e10db8.png)


因为这个函数引用的 data 是第一次的，形成了闭包，所以后面切换了户型用的也是之前的 data。

解决方式就是让 data 变成动态获取的。


![image.png](/assets/threejs/eb36bb1cd33a81aef705611412a47c39e1fe1e64.png)

```javascript
const dataRef = useRef<State['data']>(null);
dataRef.current = data;
```
```javascript
dataRef.current!.walls.forEach((item, index) => {
```
我们加了一个 ref 来保存每次渲染的 data。

计算墙可见性的时候动态从 ref 里取最新的 data。

这样切换户型之后墙的可见性的计算依然是对的：

![2025-07-08 01.07.28.gif](/assets/threejs/906b1dbbca17f86bffae70211d1734d2270d31fd.gif)

这样切换户型的问题就解决了。

## 总结

这节我们修复了两个问题。

一个是户型切换不生效，我们首先把数据清空，然后再设置新数据就好了。还有墙可见性计算的函数里用的 data 要换成动态获取的，我们用 ref 的方式来获取。

另一个是右边面板会挡住小视图，我们调整了下位置和高度。
