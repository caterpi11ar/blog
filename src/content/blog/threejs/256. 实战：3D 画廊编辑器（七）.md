这节先不着急继续开发，我们把项目部署到 github

这样后面就可以直接通过 url 访问了。

https://quarkgluonplasma.github.io/threejs-gallery-editor/

比较简单，这节讲一下怎么搞。

首先新建一个项目：


![image.png](/assets/threejs/45693399786871fd19f19c958acaa7befba1b8c7.png)


![image.png](/assets/threejs/a3e2295d0b94c7cda52fb0e6fbf4d0b562e62f07.png)

然后改一下 vite 配置

因为访问的路径是带了项目名的，所以一定要设置 base：

![image.png](/assets/threejs/ac2105afe7cf51320fb0961a251eb1c71436bc2a.png)

这样，代码里的 mp3、glb 等资源文件，就是用 /threejs-xxx/xxx.mp3 这样的路径访问了。

我们先设置 build.minify 为 false，不压缩

![image.png](/assets/threejs/b0ebd5d09829566b43d94b98b80b739d9fd96758.png)

之后进入 dist 目录

```
git init 
```
然后按照这个把代码 push 上去：


![image.png](/assets/threejs/06b9265427457fb104fcc7a737993fd08db3a4e2.png)

把 dist 下的代码复制过去，push 上去。

```javascript

git remote add origin git@github.com:QuarkGluonPlasma/threejs-gallery-editor.git
git branch -M main
git push -u origin main
```

然后在项目的 settings、pages 下设置下：


![image.png](/assets/threejs/164bf527c67fbc3961318a3cd19ab3cdec4027c6.png)

从分支直接部署，因为直接是现成的 dist 静态文件，不用 build。

选择 Deploy from a branch

之后过段时间会出现 url，这时候点击那个 url 就可以访问部署好的 3d 项目了。


![image.png](/assets/threejs/2df211bf28a91d66ea191ef0a8b4aff321b774ae.png)

如果你自己的 3d 项目做好了，也可以这样部署上去分享给别人访问。

https://quarkgluonplasma.github.io/threejs-gallery-editor/
